Note (orchestration): the debate loop (orchestration/round.py) currently calls
only argue()/rebut(); it does not invoke DebaterAgent.research(), so web search
is exercised via the standalone tool/smoke path and its unit tests, not from
inside a debate run. Wiring research() into each turn is a follow-up design
decision (it multiplies API calls and search-provider rate pressure).
